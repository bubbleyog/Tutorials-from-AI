# 第一章示例代码详解

本章提供的示例代码位于 `examples/` 目录下，旨在帮助您理解 Linux Shell 脚本和环境变量的基本概念。

## 示例 1: 基础 Shell 脚本 (`01_hello.sh`)

这个脚本展示了一个最简单的 Shell 程序结构。

### 代码解析

*   **Shebang (`#!/bin/bash`)**:
    *   位于脚本的第一行，至关重要。
    *   它告诉操作系统使用哪个解释器（这里是 `/bin/bash`）来执行后续的代码。
    *   如果没有这一行，脚本可能无法正确运行。

*   **输出 (`echo`)**:
    *   `echo "Hello..."`: 在终端打印文本。
    *   `$(date)`: 命令替换语法。它会先运行 `date` 命令，然后把结果填入这个位置。

*   **变量 (`MY_NAME="Researcher"`)**:
    *   **定义时**: 不需要 `$` 符号，且**等号两边不能有空格**（非常重要！）。
    *   **使用时**: 需要加 `$`，如 `$MY_NAME`。

*   **循环 (`for i in {1..3}`)**:
    *   Shell 中的基础循环结构，用于批量处理任务。

## 示例 2: 环境变量操作 (`02_env_vars.sh`)

环境变量是 HPC 系统中控制程序行为的关键。

### 核心变量说明

*   `$HOME`: 指向你的个人主目录 `/home/username`。所有在这个目录下的文件都属于你。
*   `$PATH`: **最重要**的环境变量。
    *   当你输入一个命令（如 `python`）时，系统会按照 `$PATH` 中列出的目录顺序依次查找。
    *   `export PATH=$HOME/my_apps/bin:$PATH`: 这是一个非常经典的操作。它将 `$HOME/my_apps/bin` 加到了 `$PATH` 的**最前面**。这意味着，如果该目录下有一个 `python`，系统会优先使用它，而不是系统默认的 `python`。

*   `OMP_NUM_THREADS`:
    *   这是 OpenMP 并行计算的标准环境变量。
    *   如果你运行一个多线程程序，设置 `export OMP_NUM_THREADS=4` 会限制它只使用 4 个 CPU 核心，这在共享节点的 HPC 环境中非常重要，可以防止你的程序占满整个节点导致干扰他人。

## 示例 3: Module 系统演示 (`03_module_demo.sh`)

这个脚本演示了 HPC 特有的软件管理逻辑。

### 为什么需要 Module？

在个人电脑上，你可能只安装一个版本的 GCC。但在 HPC 上，为了兼容旧代码和新算法，管理员可能安装了 GCC 4.8, 5.4, 8.3, 9.3, 11.2 等多个版本。

如果直接把它们都放入 `/usr/bin`，文件名冲突会非常严重。

### Module 是如何工作的？

当你执行 `module load gcc/9.3.0` 时，Module 系统在后台默默做了以下事情：
1.  修改 `PATH`，将 GCC 9.3 的 `bin` 目录加到前排。
2.  修改 `LD_LIBRARY_PATH`，将 GCC 9.3 的 `lib64` 目录加到前排。
3.  修改 `MANPATH` 等其他变量。

当你 `module unload` 时，它可以干净地撤销这些修改，让环境恢复原状。

### 关键命令回顾
*   `module avail`: 看看店里有什么货。
*   `module load`: 把货拿上架（加载到当前环境）。
*   `module list`: 看看购物车里有什么（当前加载了什么）。
*   `module purge`: 既然还没买单，那就清空购物车吧（重置环境）。
