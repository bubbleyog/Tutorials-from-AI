# 第一章：现代 C++ 高性能计算基础

在高性能计算 (HPC) 领域，性能是核心指标。传统的 C++ (C++98/03) 虽然高效，但在资源管理和避免不必要的开销方面往往需要复杂的代码技巧。现代 C++ (C++11/14/17/20) 引入了许多革命性的特性，使得我们能够在**不损失（甚至提升）性能的前提下，编写更安全、更简洁的代码**。

本章将介绍几个对 HPC 至关重要的现代 C++ 概念。

## 1. RAII 与资源管理

**RAII (Resource Acquisition Is Initialization)** 是 C++ 中最重要的编程惯用法。它的核心思想是：**资源的生命周期应该与对象的生命周期绑定**。

在 HPC 程序中，我们经常需要管理大量的内存（如大型矩阵、网格数据）。手动使用 `malloc/free` 或 `new/delete` 容易导致内存泄漏，尤其是在发生异常或复杂的控制流跳转时。

### 推荐做法
- 优先使用标准容器：`std::vector` 是管理动态数组的首选，它自动处理内存分配和释放，且保证内存连续（对缓存友好）。
- 对于独占资源，使用 `std::unique_ptr`。
- 只有在极少数需要手动管理底层内存以优化极致性能时，才封装自己的资源管理类（如本章示例中的 `Matrix` 类）。

## 2. 移动语义 (Move Semantics) —— 避免昂贵的拷贝

在 C++11 之前，如果你想从一个函数返回一个巨大的矩阵，或者将其存入容器，往往会发生**深拷贝 (Deep Copy)**。这涉及分配新内存并复制所有数据，对于 HPC 应用来说，这可能是巨大的性能杀手。

**移动语义**允许我们将资源从一个对象“转移”到另一个对象，而不是复制。这通常只是指针的交换，开销极小。

### 关键概念
- **左值 (L-value)**: 有名字、可以取地址的对象（如变量 `a`）。
- **右值 (R-value)**: 临时对象、没有名字的对象（如函数返回值、字面量）。
- **`std::move`**: 将一个左值强制转换为右值引用，告诉编译器“这个对象我不用了，你可以把它的资源拿走”。

### 性能对比示例
在示例代码 `matrix_performance.cpp` 中，我们实现了一个简单的 `Matrix` 类，并对比了拷贝构造与移动构造的性能。

- **拷贝构造**: `new double[...]` + `std::copy` (O(N) 时间复杂度)
- **移动构造**: 仅仅交换指针 (O(1) 时间复杂度)

## 3. 示例代码解析

请查看 [code/matrix_performance.cpp](./code/matrix_performance.cpp)。

### 核心代码片段

```cpp
// 移动构造函数
// noexcept 告诉编译器这个操作不会抛出异常，
// 这对于 std::vector 在扩容时选择移动而不是拷贝至关重要。
Matrix(Matrix&& other) noexcept 
    : rows_(other.rows_), cols_(other.cols_), data_(other.data_) {
    // 1. 窃取资源：直接接管 other 的 data_ 指针
    
    // 2. 置空源对象：防止 other 析构时 delete 掉我们刚偷来的数据
    other.data_ = nullptr;
    other.rows_ = 0;
    other.cols_ = 0;
}
```

### 编译与运行
确保你安装了 CMake 和 C++ 编译器。

```bash
cd code
mkdir build && cd build
cmake ..
make
./matrix_performance
```

### 预期输出
你会看到移动操作的时间几乎为 0，而拷贝操作随着矩阵大小增加而显著变慢。

```text
Matrix size: 2000x2000 (30.5 MB)
Copy Test Time: 3.6000 s
Move Test Time: 0.0000 s
Speedup (Copy / Move): infx  (实际上是几千倍到几万倍的提升)
```

## 4. `constexpr` 与编译期计算

在 HPC 中，如果某些参数（如卷积核大小、物理场维度）在编译期就能确定，使用 `constexpr` 可以让编译器进行激进的优化（如循环展开、常量折叠）。

```cpp
constexpr int DIM = 3;
// 编译器知道循环次数是 3，可能会直接展开成三条指令，消除循环开销
for (int i = 0; i < DIM; ++i) { ... }
```

## 5. 容器的选择

- **`std::vector`**: **HPC 之王**。内存连续，预取友好 (Prefetching friendly)，随机访问快。绝大多数情况下应首选。
- **`std::array`**: 栈上分配的固定大小数组。比 `vector` 少了堆分配开销，适合小型的固定大小对象（如 3D 坐标点、小矩阵）。
- **`std::list` / `std::map`**: **HPC 杀手**。节点式存储，内存不连续，造成大量的 Cache Miss。在核心计算循环中应尽量避免使用。

## 6. 总结

- 现代 C++ 通过 RAII 管理资源，杜绝内存泄漏。
- 移动语义将“深拷贝”优化为“指针交换”，大幅提升大对象传递效率。
- 始终使用 `std::vector` 作为默认容器，除非你有极强的理由使用其他容器。
