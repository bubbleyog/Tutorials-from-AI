## 🔑 核心概念预览

### 复杂度分析
> "算法效率的度量标准"

- **时间复杂度**：算法执行时间随输入规模的增长趋势
- **空间复杂度**：算法所需内存随输入规模的增长趋势

### 常见复杂度

| 复杂度 | 名称 | 示例 |
|--------|------|------|
| O(1) | 常数 | 数组访问 |
| O(log n) | 对数 | 二分查找 |
| O(n) | 线性 | 遍历数组 |
| O(n log n) | 线性对数 | 归并排序 |
| O(n²) | 平方 | 冒泡排序 |
| O(2ⁿ) | 指数 | 暴力递归 |

### 分治思想
> "分而治之"

将大问题分解为小问题，解决小问题后合并结果。

### 动态规划
> "记住过去，优化未来"

将复杂问题分解为重叠子问题，通过存储子问题的解避免重复计算。

## 📝 学习建议

1. **先理解，后编码**：理解算法思想比记住代码更重要
2. **手写推演**：在纸上画图推演算法过程
3. **分析复杂度**：每个算法都要分析时间和空间复杂度
4. **多做练习**：算法需要大量练习才能掌握
5. **循序渐进**：从简单问题开始，逐步增加难度

## ⚠️ 难度提示

本章内容是算法入门，但某些概念（如动态规划）可能需要反复学习才能真正掌握。建议：

1. 不要急于求成，一节一节稳扎稳打
2. 遇到困难的题目，先看解析，然后独立实现
3. 隔一段时间后回顾，检验掌握程度
